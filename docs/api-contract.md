# عقد الربط بين القطاعات — API & Data Contract

هذه الوثيقة تحدّد **صيغة البيانات المتبادلة** بين القطاعات الثلاثة.
الالتزام بها إلزامي، وأي تعديل يعتمده القطاع الأول (المسؤول).

مسار الملف داخل المستودع: `docs/api-contract.md`

---

## 🔄 تدفق البيانات العام

```
[الواجهة] --(1) رفع صوت--> [الخادم] --(2) استدعاء--> [محرك AI]
                                                          |
[الواجهة] <--(4) عرض المحضر-- [الخادم] <--(3) نتيجة JSON--
```

1. الواجهة ترفع الملف الصوتي للخادم.
2. الخادم يرسل الصوت لمحرك الذكاء الاصطناعي.
3. محرك AI يعيد نتيجة منظّمة (JSON) للخادم.
4. الخادم يخزّنها ويعيدها للواجهة للعرض، ويرسل المحضر بالبريد.

---

## 1️⃣ من الواجهة إلى الخادم — رفع اجتماع

**الطلب:** `POST /api/meetings`
(نوع المحتوى: `multipart/form-data`)

| الحقل | النوع | الوصف |
|-------|-------|-------|
| `title` | نص | عنوان الاجتماع |
| `audio` | ملف | الملف الصوتي (mp3 / wav) |
| `participants` | قائمة | المشاركون (اسم + بريد) |

**رد الخادم:**
```json
{
  "meeting_id": "mtg_00123",
  "status": "processing",
  "message": "تم استلام الاجتماع وجارٍ المعالجة"
}
```

---

## 2️⃣ من الخادم إلى محرك الذكاء الاصطناعي — طلب المعالجة

**الاستدعاء (دالة أو `POST /process`):**
```json
{
  "meeting_id": "mtg_00123",
  "audio_path": "/storage/mtg_00123.wav",
  "language": "ar",
  "participants": ["أحمد", "سارة", "خالد"]
}
```

- `language`: `"ar"` عربي، `"en"` إنجليزي، `"auto"` كشف تلقائي.

---

## 3️⃣ من محرك الذكاء الاصطناعي إلى الخادم — النتيجة الموحّدة ⭐

هذا هو **العقد الأهم**. يلتزم القطاع الثاني بإخراجه بهذه الصيغة تماماً:

```json
{
  "meeting_id": "mtg_00123",
  "status": "done",
  "language": "ar",
  "transcript": "النص الكامل المفرّغ للاجتماع ...",
  "summary": "ملخص موجز لأهم ما دار في الاجتماع ...",
  "decisions": [
    "اعتماد الميزانية الجديدة للربع القادم",
    "تأجيل إطلاق الحملة أسبوعين"
  ],
  "action_items": [
    {
      "task": "إعداد تقرير الميزانية",
      "assignee": "أحمد",
      "due_date": "2026-07-20",
      "priority": "high"
    },
    {
      "task": "تصميم مواد الحملة",
      "assignee": "سارة",
      "due_date": null,
      "priority": "medium"
    }
  ],
  "error": null
}
```

### شرح الحقول

| الحقل | النوع | ملاحظات |
|-------|-------|---------|
| `meeting_id` | نص | نفس المعرّف المُرسَل |
| `status` | نص | `done` أو `error` |
| `transcript` | نص | النص الكامل المفرّغ |
| `summary` | نص | الملخص |
| `decisions` | قائمة نصوص | القرارات المتخذة |
| `action_items` | قائمة كائنات | بنود العمل (انظر أدناه) |
| `error` | نص أو null | رسالة الخطأ إن فشلت المعالجة |

**كائن `action_item`:**
| الحقل | النوع | ملاحظات |
|-------|-------|---------|
| `task` | نص | وصف المهمة |
| `assignee` | نص | المسؤول (أو `"غير محدد"`) |
| `due_date` | نص أو null | تاريخ `YYYY-MM-DD` أو null |
| `priority` | نص | `high` / `medium` / `low` |

---

## 4️⃣ من الخادم إلى الواجهة — عرض المحضر

**الطلب:** `GET /api/meetings/{meeting_id}`

**الرد:** نفس بنية النتيجة أعلاه، مضافاً إليها:
```json
{
  "created_at": "2026-07-09T10:30:00",
  "pdf_url": "/api/meetings/mtg_00123/pdf",
  "participants": [
    { "name": "أحمد", "email": "ahmed@example.com" }
  ]
}
```

**قائمة كل الاجتماعات:** `GET /api/meetings`
```json
{
  "meetings": [
    { "meeting_id": "mtg_00123", "title": "اجتماع التخطيط", "status": "done", "created_at": "2026-07-09T10:30:00" }
  ]
}
```

---

## 📌 قواعد ثابتة يلتزم بها الجميع

1. جميع النصوص بترميز **UTF-8** لضمان سلامة العربية.
2. التواريخ بصيغة **ISO 8601** (`YYYY-MM-DD`).
3. أسماء الحقول بالإنجليزية و **حالة snake_case** دائماً.
4. عند الخطأ: `status = "error"` مع رسالة في `error`، وباقي الحقول `null`.
5. أي تغيير في هذا العقد يُوثّق هنا ويُعتمد من القطاع الأول قبل التطبيق.
